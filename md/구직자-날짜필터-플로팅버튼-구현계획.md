# 구직자 공고 페이지 — 날짜 필터 플로팅 버튼 구현 계획

## 유사 앱 구현 방식 참고

### 1. Google Maps — 플로팅 필터 칩
- **방식:** 지도 위에 "가격", "영업 중", "평점" 등의 필터 칩이 상단에 고정
- **UX:** 칩 클릭 → 드롭다운 또는 바텀시트, 선택 시 칩 배경색 변경 + × 버튼 등장
- **특징:** 필터 적용된 칩은 파란색, 미적용은 흰색 — 상태를 시각적으로 명확히 구분

### 2. 배달의민족 — 상단 카테고리 필터
- **방식:** 스크롤해도 상단에 고정된 필터 칩 바
- **UX:** 칩 선택 시 즉시 리스트 업데이트, 선택된 칩은 강조색
- **특징:** 매우 가볍고 빠른 필터링 경험

### 3. Airbnb — 날짜 필터
- **방식:** 상단 검색바에 날짜 클릭 → 인라인 달력 확장 → 날짜 선택 → 리스트 즉시 업데이트
- **UX:** 날짜 선택 중에도 결과가 흐리게(dim) 표시되며 실시간 업데이트 예고
- **특징:** 달력에서 날짜별 최저가 표시 (우리 버전: 날짜별 공고 수 표시 가능)

### 4. 카카오맵 — 플로팅 조건 버튼
- **방식:** 우하단 플로팅 "조건 설정" 버튼 → 클릭 시 바텀시트 슬라이드업
- **UX:** vaul 스타일의 드래그 가능한 바텀시트
- **특징:** 버튼에 현재 적용 중인 조건 수 뱃지 표시

### 5. Naver 부동산 — 필터 고정 바
- **방식:** 지역 선택 + 날짜 선택이 상단 스티키 필터 바에 나란히 배치
- **UX:** 지역 선택(현재 존재)과 날짜 선택을 같은 레벨로 배치

**채택 방향:**
- **기본 상태:** 구직자에게 "일하고 싶은 날짜를 선택하세요" 플로팅 버튼 (카카오맵 스타일)
- **날짜 선택 후:** 버튼이 "4월 11일 근무 공고 N건" 으로 변경 + × 버튼
- **달력 UI:** vaul 바텀시트 + react-day-picker (둘 다 이미 설치됨)
- **필터링:** 서버사이드 API 필터링 (region 필터와 동일한 방식)

## 플로팅 버튼 상세 설계

### 위치 및 형태

```
위치: fixed bottom-24 left-1/2 -translate-x-1/2   (화면 하단 중앙)
형태: pill 버튼 (둥근 캡슐 모양)
크기: px-5 py-3, min-w-48
그림자: shadow-lg

[미선택 상태]
  🗓️  일하고 싶은 날짜를 선택하세요
  배경: bg-gray-900  텍스트: text-white

[날짜 선택 후]
  📅  4월 11일 근무 공고  ×
  배경: bg-blue-600  텍스트: text-white
  우측: × 아이콘 클릭 시 필터 초기화
```

### 스크롤 연동 (UX 개선)

```
페이지 최상단: 플로팅 버튼 표시
스크롤 다운: 버튼 유지 (항상 접근 가능)
스크롤 업 중 (빠르게): 버튼 살짝 위로 올라가는 애니메이션
  → CSS: transition-transform duration-300

이유: 공고 목록이 길 때 중간에서도 날짜 필터를 바꿀 수 있어야 함
```

### 사장님 CTA와 공존

```
[사장님 CTA]  fixed bottom-6 right-4    (우하단)
[날짜 버튼]   fixed bottom-6 left-1/2   (하단 중앙)

→ 두 요소가 겹치지 않도록 위치 분리
→ 구직자(worker role)가 아닌 경우: 날짜 플로팅 버튼 미표시
   (비로그인 또는 employer에게는 "사장님 CTA"만 표시)
```

```
  │                                  │
  │  [달력 컴포넌트]                 │
  │   ← 3월         4월 →           │
  │  일 월 화 수 목 금 토            │
  │  ·  ·  ·  ·  ·  ·  ·           │
  │  (날짜들, 공고 있는 날짜 dot 표시)│
  │                                  │
  │  선택됨: 4월 11일 ~ 4월 12일    │
  │  [이 날짜로 공고 보기]           │
  └─────────────────────────────────┘
```

### 달력 세부 UX

```
공고 있는 날짜 표시:
  → 날짜 아래 파란 dot (·) 표시
  → API: GET /api/jobs/available-dates?region=서울
    → 해당 지역에 active 공고가 있는 날짜 배열 반환
    → ["2025-04-11", "2025-04-12", "2025-04-15"] 형태

선택 범위 (구직자: 최대 5일 연속):
  → 달력-날짜선택-구현계획.md의 WorkerDateRangePicker 방식 적용
  → 시작일 클릭 → 끝일 클릭 → 범위 하이라이트

선택 초기화:
  → 바텀시트 상단 "초기화" 버튼 (우상단)
```

### 필터 활성 상태 표시

```
필터 바 아래에 "활성 필터" 칩 표시:
  ┌─────────────────────────┐
  │ 📅 4월 11일 ~ 4월 12일  ×│
  └─────────────────────────┘

+ 결과 수 표시:
  "4월 11~12일 근무 공고 7건"  (공고 그리드 위에)
```

### 날짜 필터 적용 시 공고 없을 때

```
기존: "등록된 공고가 없습니다 · 다른 지역을 선택하거나 나중에 다시 확인해보세요"

날짜 필터 적용 후 없을 때:
  "4월 11일에 [서울] 공고가 없어요"
  "날짜를 변경하거나 지역을 바꿔보세요"
  [날짜 다시 선택하기] 버튼  ← 바텀시트 재오픈
```

## 데이터 흐름 & API

### 필터링 방식: 서버사이드 (권장)

```
현재: GET /api/jobs?region=서울
변경: GET /api/jobs?region=서울&startDate=2025-04-11&endDate=2025-04-12

API 내부 (route.ts):
  const startDate = searchParams.get("startDate")
  const endDate   = searchParams.get("endDate")

  where: {
    status: "active",
    ...(region ? { location: { contains: region } } : {}),
    ...(startDate && endDate ? {
      dates: {
        hasSome: getDatesInRange(startDate, endDate)
        // ["2025-04-11", "2025-04-12"] 중 하나라도 포함
      }
    } : {})
  }
```

> 이 쿼리는 달력-날짜선택-구현계획.md의 `dates: String[]` 스키마 변경 이후 동작

### 공고 있는 날짜 조회 API (달력 dot 표시용)

```
GET /api/jobs/available-dates?region=서울
  → 해당 지역의 active 공고 dates 배열을 flatten + unique
  → 응답: { dates: ["2025-04-11", "2025-04-12", "2025-04-15"] }
  → 달력에서 이 날짜들에 dot 표시
```

## 애니메이션 상세

```
FloatingDateButton:
  - 페이지 로드 시: translateY(100%) → translateY(0), 0.4s ease-out, delay 0.5s
  - 날짜 선택 완료 시: 버튼 배경 gray-900 → blue-600, 0.2s transition
  - × 클릭 시: blue-600 → gray-900, 텍스트 복구

DateFilterDrawer (vaul):
  - 기본 vaul 슬라이드업 애니메이션 사용
  - snapPoints: [0.5, 1] → 절반 / 전체화면 두 가지 스냅

ActiveFilterChip:
  - 날짜 선택 시: opacity 0 → 1, translateY(-4px) → 0, 0.2s
  - 초기화 시: opacity 1 → 0, 0.15s
```

## 주요 UX 결정 사항 요약

| 항목 | 결정 | 이유 |
|------|------|------|
| 필터 진입점 | 플로팅 버튼 + sticky 필터 바 두 곳 | 스크롤 위치 관계없이 접근 가능 |
| 달력 표시 방식 | vaul 바텀시트 | 이미 설치됨, 모바일 친화적 |
| 필터링 시점 | "이 날짜로 공고 보기" 버튼 클릭 시 | 날짜 선택 중 불필요한 API 호출 방지 |
| 공고 dot 표시 | 공고 있는 날짜에만 파란 점 | 사용자가 빈 날짜 선택하는 헛수고 방지 |
| 필터 초기화 | × 버튼 1번 클릭 | 마찰 최소화 |
| 필터링 방식 | 서버사이드 API | region 필터와 일관성, 대용량 대응 |
