# 달력 날짜 선택 UI 구현 계획

## 유사 앱 구현 방식 참고

### 1. Airbnb — 연속 범위 선택 (Range Picker)
- **방식:** 시작일 클릭 → 끝일 클릭, 그 사이 전체 하이라이트
- **UX 포인트:** 호버 시 미리보기 하이라이트, 시작/끝 날짜에 둥근 캡(cap), 중간 날짜는 연한 색 채움
- **제약:** min/max date 설정으로 선택 불가 날짜 회색 처리

### 2. Wonolo / Instawork (미국 일용직 매칭 앱) — 개별 시프트 선택
- **방식:** 달력에서 날짜 탭 → 해당 날짜의 시프트 목록 표시 → 시프트 선택
- **UX 포인트:** 이미 마감된 날짜 비활성화, 선택된 날짜 강조, 주간 뷰 기본
- **구직자 관점:** 자신이 가능한 날짜를 개별 탭핑으로 표시

### 3. Booking.com — 비연속 다중 선택 (Flexible Dates)
- **방식:** "유연한 날짜" 모드에서 특정 요일이나 날짜들을 체크박스처럼 선택
- **UX 포인트:** 선택된 각 날짜에 파란 원 표시, 상단에 "N일 선택됨" 카운터
- **사장 관점에서 참고 가능:** 비연속 다중 선택 + 카운터

### 4. 당근마켓 이웃알바 — 단순 날짜 범위
- **방식:** "시작일 ~ 종료일" 텍스트 입력 또는 간단한 달력
- **특이점:** 반복 근무(요일 선택)와 단기 근무(날짜 선택) 분리

## DB 스키마 변경

현재 `JobPosting.date`는 `String` 단일 값 → **다중 날짜** 지원으로 변경 필요.
현재 `Application`에는 날짜 필드 없음 → **구직자 신청 날짜** 추가 필요.

```prisma
model JobPosting {
  // 기존 date String → 아래로 교체
  dates     String[]   // ["2025-04-01", "2025-04-03", "2025-04-07"] 비연속 허용
  // ...나머지 필드 유지
}

model Application {
  // 기존 필드 유지 + 아래 추가
  selectedDates String[]  // ["2025-04-01", "2025-04-02"] 구직자가 신청한 연속 날짜
}
```

마이그레이션:
```bash
npx prisma migrate dev --name update-dates-to-array
```

## 구직자용 달력 — 연속 범위 선택기

### UX 플로우 (Airbnb Range Picker 참고)

```
1. 달력 팝업 열기
2. 날짜 클릭 → 시작일(anchor) 설정, 파란 원으로 표시
3. 다른 날짜 hover → 시작일~hover일 미리보기 하이라이트
4. 두 번째 클릭 → 끝일 확정
   - 끝일이 시작일보다 이전이면 → 시작일로 리셋 후 재선택
   - 끝일이 시작일 + 4일 초과 → 자동 클램프 (시작일 + 4일로 고정)
5. 선택 범위 확정: 시작일 ~ 끝일 하이라이트
   - 시작일: 파란 원, 오른쪽 반 파란 배경
   - 중간일: 연한 파란 배경 (#EFF6FF), 날짜 텍스트 진하게
   - 끝일: 파란 원, 왼쪽 반 파란 배경
6. 확인 버튼 클릭 → 선택 완료
```

### 선택 제약 로직

```
MIN_DATE = 내일 (today + 1)
MAX_DATE = 내일부터 5일 후 (today + 6, exclusive)
MAX_RANGE = 5일

- MIN_DATE 이전: 선택 불가 (회색 + 클릭 무시)
- MAX_DATE 이후: 선택 불가 (회색 + 클릭 무시)
- 선택된 범위가 5일 초과: 끝일을 startDate + 4로 클램프
- 비연속(gap) 선택: 불가 (UI 자체가 범위 선택이므로 자연히 방지됨)
```

### 상태(State) 설계

```typescript
type RangeState = {
  startDate: Date | null
  endDate: Date | null
  hoverDate: Date | null   // hover 미리보기용
}

// 날짜가 범위 내인지 판단
const isInRange = (date: Date, start: Date | null, end: Date | null) =>
  start && end && date >= start && date <= end

// 날짜가 선택 가능한지 판단
const isSelectable = (date: Date) => {
  const tomorrow = addDays(today, 1)
  const maxDate = addDays(today, 5)   // 내일 포함 5일 → 오늘+5가 최대끝일
  return date >= tomorrow && date <= maxDate
}
```

### 달력 셀 스타일 결정 로직

```
각 날짜 셀에 대해:

isStart → bg-blue-600 rounded-l-full text-white
isEnd   → bg-blue-600 rounded-r-full text-white
isMiddle → bg-blue-50 text-blue-900
isHoverRange → bg-blue-50/50 (미확정 hover 미리보기)
isDisabled → text-gray-300 cursor-not-allowed
isToday → underline or dot
```

### 컴포넌트 구조

```
<WorkerDateRangePicker>
  ├── CalendarHeader (월 이름, 이전/다음 월 버튼)
  ├── WeekdayLabels (일 월 화 수 목 금 토)
  └── CalendarGrid
      └── CalendarCell × 28~42개
          (날짜, 선택상태에 따라 스타일 결정)
  └── FooterBar
      ├── "최대 5일 연속 선택 가능" 안내 텍스트
      ├── 선택된 날짜 요약 (예: 4월 11일 ~ 4월 12일, 2일)
      └── 확인 버튼
```

## 사장님용 달력 — 비연속 다중 선택기

### UX 플로우 (Booking.com Flexible Dates 참고)

```
1. 달력 팝업 열기 (2개월 뷰: 이번달 + 다음달)
2. 날짜 클릭 → 선택 토글 (선택됨 ↔ 해제됨)
   - 선택됨: 파란 원 + 흰 텍스트
   - 해제됨: 기본 스타일
3. 이미 31일 선택된 상태에서 추가 클릭 → 토스트 메시지 "최대 31일까지 선택 가능합니다"
4. 상단에 "N일 선택됨" 카운터 실시간 업데이트
5. 확인 버튼 클릭 → 선택 완료
```

### 선택 제약 로직

```
MIN_DATE = 내일 (today + 1)
MAX_DATE = today + 31
MAX_COUNT = 31일

- MIN_DATE 이전: 선택 불가
- MAX_DATE 이후: 선택 불가
- 31일 초과 선택 시도: 무시 + 안내 메시지
- 비연속 선택: 허용 (토글 방식)
```

### 상태(State) 설계

```typescript
type MultiSelectState = {
  selectedDates: Set<string>   // "2025-04-01" 형태의 날짜 문자열 Set
}

// 날짜 토글
const toggleDate = (dateStr: string) => {
  setSelected(prev => {
    const next = new Set(prev)
    if (next.has(dateStr)) {
      next.delete(dateStr)
    } else {
      if (next.size >= 31) { showToast("최대 31일"); return prev }
      next.add(dateStr)
    }
    return next
  })
}
```

### 달력 셀 스타일 결정 로직

```
isSelected → bg-blue-600 rounded-full text-white font-semibold
isDisabled → text-gray-300 cursor-not-allowed
isToday    → ring-1 ring-blue-400
```

### 컴포넌트 구조

```
<EmployerMultiDatePicker>
  ├── Header
  │   ├── "N일 선택됨" 카운터 (파란 뱃지)
  │   └── 전체 초기화 버튼
  ├── MonthSection (이번 달)
  │   ├── MonthLabel + 네비게이션
  │   ├── WeekdayLabels
  │   └── CalendarGrid × 셀
  ├── MonthSection (다음 달)
  │   └── (동일 구조)
  └── FooterBar
      ├── 선택된 날짜 미리보기 칩 (최대 5개 + "외 N일")
      └── 확인 버튼
```

## 공통 고려 사항

### 라이브러리 선택
외부 라이브러리 없이 **커스텀 구현** 권장:
- `date-fns` 이미 설치됨 → 날짜 계산 유틸로 활용
- 외부 달력 라이브러리(react-datepicker 등)는 스타일 커스터마이징 난이도 높음
- 필요 기능이 명확하므로 직접 구현이 더 가볍고 빠름

### date-fns 활용 함수
```typescript
import {
  addDays, startOfMonth, endOfMonth,
  startOfWeek, eachDayOfInterval,
  format, isBefore, isAfter,
  isSameDay, isSameMonth
} from "date-fns"
import { ko } from "date-fns/locale"
```

### 모바일 UX
- 모바일 기본: 바텀시트(bottom sheet) 형태로 달력 표시
- 터치 타겟: 각 날짜 셀 최소 44×44px
- 스와이프로 월 이동 (선택 사항)

### 접근성
- `aria-selected`, `aria-disabled` 속성 추가
- 키보드 네비게이션 (방향키로 날짜 이동)은 추후 고려

## 구현 순서

```
1. [스키마] date → dates[], selectedDates[] 마이그레이션
2. [컴포넌트] WorkerDateRangePicker 구현 (범위 선택 로직 + 스타일)
3. [컴포넌트] EmployerMultiDatePicker 구현 (토글 다중 선택 로직 + 스타일)
4. [연결] /employer/post Step 2에 EmployerMultiDatePicker 적용
5. [연결] /jobs 페이지 지원 흐름에 WorkerDateRangePicker 모달 적용
6. [API] jobs POST/GET, apply POST에서 날짜 배열 처리 로직 수정
7. [UI] 공고 카드에 날짜 배열 표시 방식 정리 (예: "4/1 ~ 4/3, 4/7" 형태)
```
