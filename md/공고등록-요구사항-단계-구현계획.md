# 공고 등록 — 요구사항 단계 추가 구현 계획

## 유사 앱 구현 방식 참고

### 1. Wonolo (미국 일용직 앱)
- **방식:** 공고 등록 마지막에 "Job Requirements" 섹션
- **UX:** 카테고리별 칩(chip) 선택 + 체크리스트
- **예시 태그:** "Steel-toed boots required", "Able to lift 50 lbs", "Must have forklift cert"
- **특징:** 선택지는 고정이지만 자유 입력도 허용

### 2. 사람인/잡코리아 (국내 채용 플랫폼)
- **방식:** "우대사항", "자격조건" 섹션을 필드 그룹으로 분리
- **UX:** 각 카테고리를 아코디언으로 펼쳐서 체크박스 선택
- **특징:** 직무별 자동 추천 (선택한 업종에 맞는 자격증 자동 제안)

### 3. Indeed (글로벌 채용 플랫폼)
- **방식:** "Add job requirements" → 단계별 질문 형식
- **UX:** "Is experience required?" → Yes/No → Yes면 "How many years?" 단계적 전개
- **특징:** 질문에 답하면서 자연스럽게 JD 구성됨 (Conversational UI)

### 4. 당근마켓 이웃알바
- **방식:** 단순 토글 체크박스 ("초보 가능", "경력자 우대", "여성 우대" 등)
- **UX:** 매우 단순, 3~4개 옵션만

**채택 방향:** Wonolo의 칩 선택 방식 + Indeed의 단계별 질문 형식 조합
→ 사장님이 "질문에 답하는 느낌"으로 진행하면서 체크리스트를 완성

## 현재 공고 등록 Step 구조

```
Step 1: 기본정보 (업종, 사업장명, 위치)
Step 2: 근무조건 (날짜, 시간, 인원, 급여)
Step 3: 업무내용 (업무 태그, 상세 설명) + 공고 요약 + 등록 버튼
```

## 변경 후 Step 구조

```
Step 1: 기본정보
Step 2: 근무조건
Step 3: 업무내용 (업무 태그, 상세 설명)
Step 4: 요구사항 ← 신규 추가 (등록 버튼 여기로 이동)
```

상단 진행 바도 4단계로 변경 (`[1,2,3,4].map(...)`)

## Step 4 세부 구성

### 질문 목록 및 선택지

```
Q1. 이 일을 위해 경험이나 자격이 필요한가요?
    → [선택 없음 (누구나 가능)] 또는 아래에서 복수 선택

    ✓ 초보 가능         ✓ 제조업 경험자 우대
    ✓ 물류 경험자 우대  ✓ 건설 현장 경험자
    ✓ 지게차 자격증     ✓ 용접 자격증
    ✓ 운전면허 1종      ✓ 식품위생 교육 이수
    ✓ 위험물 취급 면허  + [직접 입력]

Q2. 출근 시 가져와야 할 것이 있나요?
    → [특별히 없음] 또는 아래에서 복수 선택

    ✓ 신분증 필수       ✓ 안전화
    ✓ 운동화            ✓ 작업복
    ✓ 장갑              ✓ 마스크/방진구
    ✓ 공구(개인 지참)   + [직접 입력]

Q3. 신체 조건이 있나요?
    → [특별히 없음] 또는 아래에서 복수 선택

    ✓ 장시간 서서 작업 가능    ✓ 중량물 취급 가능 (20kg↑)
    ✓ 야외/외부 작업 가능      ✓ 교대 근무 가능
    ✓ 고소 작업 가능           ✓ 야간 작업 가능

Q4. 연령대 제한이 있나요?  (법적 고지: 연령 차별 주의)
    (단일 선택)
    ○ 무관   ○ 20~40대 선호   ○ 40~60대 선호   ○ 직접 입력
```

## UX 플로우 상세

### 단계 진입 애니메이션
```
Step 3 → Step 4 전환 시:
  - 헤더: "마지막 단계예요!" 로 변경
  - 서브텍스트: "구직자가 미리 확인하면 미스매치를 줄일 수 있어요"
  - 각 질문 카드가 아래에서 위로 fade-in (stagger 0.1s 간격)
```

### 각 질문 카드 구조
```
┌─────────────────────────────────────────────────┐
│  Q1. 경험/자격  ·  선택 안 하면 "누구나 가능"  │
│                                           [선택 N] │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │ 초보 가능│ │제조업 경험│ │지게차 자격│         │
│  └──────────┘ └──────────┘ └──────────┘         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │운전면허  │ │ 용접 자격 │ │+ 직접입력│         │
│  └──────────┘ └──────────┘ └──────────┘         │
└─────────────────────────────────────────────────┘
```

### "누구나 가능 / 특별히 없음" 처리
```
- 선택지를 아무것도 선택하지 않은 상태 = "누구나 가능" 자동 적용
- Q2~3에서 아무것도 선택하지 않으면 공고 카드에 해당 섹션 미표시
- 모든 질문은 선택 사항 (스킵 가능), 강제 입력 X
```

### 직접 입력 칩
```
"+ 직접 입력" 클릭 시 인라인 텍스트 입력 필드 등장
→ Enter/확인 → 커스텀 칩으로 추가
→ 최대 3개 직접 입력 허용
```

### 공고 요약 프리뷰 (Step 4 하단)
```
기존 Step 3에 있던 "📋 공고 요약" 카드를 Step 4로 이동
→ requirements 선택 내용도 함께 표시
→ "안전화, 신분증 지참 필요 · 지게차 자격증 우대" 형태로 한 줄 요약
```

## 상태(State) 설계

```typescript
type Requirements = {
  skills: string[]       // Q1: 경험/자격
  items: string[]        // Q2: 준비물
  physical: string[]     // Q3: 신체 조건
  ageRange: string       // Q4: 연령 (단일 선택)
  customSkills: string[] // 직접 입력 (skills)
  customItems: string[]  // 직접 입력 (items)
}

// form state에 추가
const [form, setForm] = useState({
  // ...기존 필드들...
  requirements: {
    skills: [],
    items: [],
    physical: [],
    ageRange: "무관",
    customSkills: [],
    customItems: [],
  } as Requirements
})
```

## DB 스키마 변경

`JobPosting` 모델에 `requirements` JSON 필드 추가:

```prisma
model JobPosting {
  // ...기존 필드...
  requirements Json?   // Requirements 객체 저장
}
```

저장 형태 예시:
```json
{
  "skills": ["초보 가능", "제조업 경험자 우대"],
  "items": ["안전화", "신분증 필수"],
  "physical": ["중량물 취급 가능"],
  "ageRange": "무관",
  "customSkills": ["3D 프린터 운용 경험"],
  "customItems": []
}
```

마이그레이션:
```bash
npx prisma migrate dev --name add-job-requirements
```

## 공고 카드 표시 (구직자 /jobs 페이지)

### 요구사항 섹션 추가

```
기존 카드 구조:
  [회사/카테고리] → [날짜/시간/급여] → [업무태그] → [지원버튼]

변경 후:
  [회사/카테고리] → [날짜/시간/급여] → [업무태그]
  → [요구사항 요약] ← 신규
  → [지원버튼]
```

요구사항 요약 표시:
```
준비물: 안전화, 신분증   ← items 있을 때만 표시
우대: 지게차 자격증      ← skills에 경험/자격 있을 때만 표시
초보 가능                ← skills에 "초보 가능" 포함 시 초록 배지
```

### 공고 상세 모달 (추후 확장)
```
카드 클릭 → 상세 모달
  → requirements 전체 표시
  → "신체 조건: 중량물 취급 가능 (20kg↑)"
  → "준비물: 안전화 필수, 신분증 필수"
```

## API 변경

```
POST /api/jobs  (route.ts)
  body에 requirements 추가:
  const { ..., requirements } = body
  prisma.jobPosting.create({
    data: { ..., requirements: requirements ?? null }
  })

GET /api/jobs (route.ts)
  응답에 requirements 포함 (자동 포함됨, 별도 작업 없음)
```

## 구현 순서

```
1. [스키마] requirements Json? 필드 migration
2. [API] POST /api/jobs에 requirements 저장 로직 추가
3. [UI] employer/post Step 4 컴포넌트 구현 (질문 카드 4개)
4. [UI] 진행 바 3단계 → 4단계 변경
5. [UI] 제출 버튼 Step 3 → Step 4로 이동
6. [UI] Step 4에 공고 요약 프리뷰 이동
7. [UI] /jobs 공고 카드에 요구사항 요약 배지 추가
```
